# Facilitator Guide

How to run the Copilot Cowork Dojo as a workshop. Designed for groups of 6–24 knowledge workers.

## Outcomes

By the end of the workshop, every participant should:

1. Run the **CLARIFY → PLAN → DRAFT → REVIEW → REFINE → SHIP → LEARN** pipeline on a real deliverable from their own work.
2. Recognize and apply at least 4 core kata.
3. Have logged at least one lesson in `tasks/lessons.md`.
4. Know where the sensitivity line is in their context.

## Formats

- **Half-day (4h)** — see [agendas/half-day.md](./agendas/half-day.md). Best for first exposure.
- **Multi-week** — one 90-minute session per week for 4 weeks, each week one belt.
- **Office hours** — recurring 60-minute drop-in for teams already mid-flight.

## Setup checklist (T-1 week)

- [ ] Confirm participants have Microsoft 365 Copilot Cowork access.
- [ ] Ask each participant to bring **one real deliverable** they owe in the next 7 days.
- [ ] Confirm an HR-cleared workspace for any session that may touch personal data.
- [ ] Send participants the [README](../README.md) and [governance](../governance/) folder as pre-read.

## Setup checklist (T-1 day)

- [ ] Print or share the prompt library ([prompts/library.md](../prompts/library.md)) for reference.
- [ ] Pre-pin the [.github/copilot-instructions.md](../.github/copilot-instructions.md) in the demo workspace.
- [ ] Have a blank `tasks/lessons.md` ready to fill live.

## Ground rules for the room

1. Real work only. No toy examples.
2. We respect the sensitivity gate. If unsure, escalate — even in the workshop.
3. Verify before you ship. The facilitator will not let "good enough" graduate.
4. Lessons over praise. Logged failures > unlogged successes.

## Facilitator anti-patterns

- Lecturing the framework. Run the pipeline, don't describe it.
- Letting one participant dominate. Pair people up.
- Skipping the LEARN step. The dojo dies without it.
- Demo data instead of real work. Real friction is the lesson.
