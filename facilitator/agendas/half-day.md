# Half-Day Agenda (4 hours)

For a first-exposure workshop, 6–24 participants, one facilitator + one co-facilitator.

## Pre-work (sent 3 days before)

- Read the [README](../../README.md) and the six core kata.
- Bring **one real deliverable** owed in the next 7 days (memo, recap, brief, status update — not a slide deck full of charts).
- Skim [governance/data-handling.md](../../governance/data-handling.md).

---

## Block 1 (30 min) — Frame the dojo

- Why we're here: untrained Copilot use → trained Copilot coworkers.
- The mandatory pipeline (live walkthrough on facilitator's deliverable).
- The six core kata in 5 minutes.
- Ground rules.

## Block 2 (60 min) — Run the pipeline, part 1

Pair participants. Each pair picks one deliverable to run end-to-end.

1. CLARIFY — restate, surface assumptions, confirm. (15 min)
2. PLAN — sketch deliverable structure. (15 min)
3. DRAFT — section-by-section first cut with grounded context. (30 min)

Facilitator circulates, refuses to let pairs skip steps.

## Break (15 min)

## Block 3 (60 min) — Run the pipeline, part 2

4. REVIEW — five-lens critique. (20 min)
5. REFINE — targeted follow-ups. (20 min)
6. SHIP — final QA gate. Sensitivity check. Citations check. (20 min)

End of block: each pair has a shippable deliverable.

## Break (10 min)

## Block 4 (45 min) — Sensitivity, governance, multi-agent

- Walkthrough: the four-question gate on a realistic edge case. (15 min)
- Walkthrough: agent handoff with a structured packet. (15 min)
- Q&A on org-specific governance. (15 min)

## Block 5 (20 min) — LEARN

- Each participant logs at least one lesson in `tasks/lessons.md` live.
- Group discussion: which lessons would already qualify for promotion?
- Set the team's next two-week cadence (office hours, lesson review, scenario rotation).

## Close (10 min)

- Belt awards (informal — `clarify-the-ask`, `verify-before-send`, etc.).
- Commitments: each participant names one skill they'll apply tomorrow.
- Follow-up office hours scheduled.
